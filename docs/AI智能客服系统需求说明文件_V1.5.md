# 项目概述

## 1.1 项目背景

公司主营业务涵盖ICT基础设施服务、企业数字化转型服务及人力资源劳务派遣三大板块。随着业务规模扩大及数字化服务升级，现有客服体系面临渠道分散、响应时效不足、服务标准不统一等问题，亟需建设统一的AI智能客服系统，实现全渠道智能化服务覆盖。

## 1.2 建设目标

• 统一服务入口：整合官网、电商商城、社交媒体等全渠道客户咨询入口

• 7×24小时响应：AI员工自动处理80%以上常规咨询，人工客服介入处理复杂场景

• 业务精准匹配：基于AI员工配置的专业知识库与回复策略，实现不同渠道/场景差异化服务

• 数据驱动决策：采集全渠道服务数据，支撑服务优化与客服效能管理

• 智能客服分配：复杂咨询自动识别并精准分配至对应客服，支持多种分配策略配置化

## 1.3 核心设计思路

• AI员工：一个可独立配置、独立运行的AI客服实例，拥有专属的知识库、回复策略、服务话术

• 账号覆盖：每个AI员工可绑定一个或多个渠道账号（如抖音号A、小红书号B、官网C），实现分账号差异化服务

• 客服分配：当AI员工无法处理或触发转人工条件时，按配置策略将对话分配给人工客服

• 策略配置化：AI员工的回复策略、转人工规则、客服分配规则均支持后台可视化配置

# AI员工模型设计

AI员工是系统的核心管理单元，代表一个可独立对外提供服务的AI客服实例。

## AI员工基础属性

| **属性** | **是否必填** | **说明** | **示例** |
| --- | --- | --- | --- |
| 员工名称 | 必填  | 对外展示名称 | 小倩（ICT顾问） |
| 头像  | 必填  | 对话窗口展示头像 | 企业Logo或自定义形象 |
| AI 接待时间 | 必填  | 全天候接待 | 自定义时间：周一18:00～周二9:00 |
| 开场白消息 | 必填  | 首次对话欢迎语 | 您好，我是ICT服务顾问小倩，请问有什么可以帮您？ |
| AI 接待风格 | 选填  | 亲切，热情，专业，靠谱 | \- 回复时可适当加入表情符号语气词（啦~呢~哦~）、生活化语言（咱、哈、嘞、哦、对的）  <br>\- 称呼： 统一称呼客户为“老板”  <br>\- 遇到常见问题时，从多套回复模板中随机选择，避免重复感  <br>\- 当询问客户联系方式时，可以顺便补充一些诱导话术，例如：方便留个联系方式吗？我可以给您发详细的配置表、最新报价单，有优惠活动时第一时间通知您～ |
| AI 回复消息字数 | 必填（单选） | 简短/适中/详细 | 简短：单条消息20字以内  <br>适中：单条消息约20-30字  <br>详细：单条消息约60-100字 |
| AI 回复内容检查 | 选填  | v、绿泡泡、薇（因抖音、小红书等新媒体平台有风控，需用其他词语或符号替代“微信”这个词语） | 当需要回复“微信”时，替换为：{V} |
| 客户消息聚合间隔 | 选填  | 1-10秒 | 例如设置5秒，若客户连续发送多条消息，则AI会等待客户发完这组消息后，直到5秒后没有新的客户消息，AI再进行回复（系统会将客户的连续多条消息整合后输入给AI，AI会结合上下文回复一条消息） |
| AI 延迟回复间隔 | 选填  | 1-10秒 | 例如设置3秒，当客户发送完消息后，AI会等待3秒后再进行回复 |

## 回复策略配置

回复策略定义AI员工在不同场景下的回复行为，支持可视化配置。

| **策略维度** | **是否必填** | **配置项** | **说明** |
| --- | --- | --- | --- |
| 问候策略 | 选填  | 首次接入欢迎语、二次接入欢迎语、离线留言提示 | 区分新客/老客/非服务时段 |
| 应答策略 | 选填  | 知识库匹配优先级、大模型兜底开关、回复长度限制 | 优先查知识库，无匹配时走大模型生成 |
| 异常策略 | 选填  | 敏感词过滤、负面情绪安抚、无法回答时的道歉话术 | 避免负面体验 |
| 【收集】技能 | 选填  | AI会结合设置的内容项主动发问，并从客户的回复中收集信息，存储到顾客名片中 | 例如：招聘岗位、招聘人数、时间要求 |
| 【筛选】技能 | 选填  | 根据筛选目标，多轮询问，筛选出目标群体（筛选之后再获线可以获得更精准的客资） | 示例：  <br>目标定义：所在城市为上海的顾客 |
| 【排除】技能 | 选填  | 排除明显不需要的顾客，AI将不进行接待服务 | 示例：广告推销 |
| 【留资】技能 | 必填（单选） | 1、发名片：发送企业的客服名片，让客户自行添加（比如企微名片卡，必须在新媒体平台配置名片卡）  <br>2、问名片：AI主动询问客户的联系方式（若在新媒体平台配置留资卡片，则AI发送留资卡消息；若未配置，则AI发送留资话术消息） | 选项：主动发名片、问名片 |
| 【沉默追问】技能 | 选填  | 顾客沉默一段时间后，发送追问话术（获线成功后不再触发） | 示例：间隔30s，固定追问“老板，还在吗？”  <br>间隔60s，AI智能追问 |
| 【调用】技能 | 选填  | 结合企业上传的知识库，精准解答客户咨询的问题 |     |

## 转人工规则配置

定义AI员工在何种情况下将对话转交人工客服处理，支持多条件组合配置。

|     |     |     |
| --- | --- | --- |
| **触发条件** | **配置示例** | **执行动作** |
| 关键词触发 | 客户提及「投诉」「退款」「违约」「法律」 | 立即转人工+标记紧急 |
| 情感触发 | 情感分析愤怒/焦急 | 立即转人工+触发安抚话术 |
| 用户主动要求 | 客户发送「转人工」「找客服」「不要机器人」 | 立即转人工+记录转人工原因 |
| 时间触发 | 非服务时段（如晚间22:00后） | 提示留言+次日客服跟进 |

## 客服分配策略配置

当AI员工触发转人工后，系统按配置的分配策略将对话/工单分配给人工客服。支持多种策略可视化配置。

| **分配策略** | **策略说明** |
| --- | --- |
| 指定分配 | 固定分配给某一位客服 |
| 轮询分配 | 按客服列表顺序轮流分配 |
| 负载均衡 | 分配给当前待处理量最少的客服 |
| 历史归属 | 老客户优先分配给上次接待的客服 |

## 覆盖账号范围配置

每个AI员工可绑定一个或多个平台账号，实现「一个AI员工管理多个账号」或「多个AI员工分管不同账号」的灵活模式。

|     |     |     |
| --- | --- | --- |
| **配置项** | **说明** | **示例** |
| 平台类型 | 渠道平台 | 官网、抖音、小红书、微信公众号、电商商城 |
| 账号  | 平台内的唯一账号标识 | 已接入的账号选择：小红书、抖音、视频号等 |
| 账号状态 | 是否启用AI员工接管 | 启用/停用 |

- 覆盖模式示例：

一AI对多账号AI员工「小通（ICT顾问）」同时覆盖：官网客服窗口、抖音企业号、微信公众号 → 统一ICT知识库与回复策略

# 客服IM对话聊天管理

对话包含三个板块:对话导航、对话列表、对话内容、顾客信息

## 对话导航

- 导航包含：全部对话、AI接管、待介入(转人工)、未分配、未回复
- 我的：我接待的/AI接管/待我介入
- 同事对话：权限范围内同事正在接待的对话，适合协助回复
- 接入渠道：可分渠道查看对话

## 对话列表

- 对话列表显示用户名、头像、最近聊天内容，时间（参考微信聊天列表）

## 对话内容

- 点击对话列表中的对话，进入到对话详情页面，支持手动转接对话和结束对话，可以编辑文字，表情、图片，发送名片等。

## 防冲突与合规机制

- 撞单防护：同一客户30天内已有活跃会话时，新咨询自动关联至原客服，避免重复跟进
- 客户隐私：客服仅可见分配给自己的会话信息，无权查看其他客服客户数据

# 非功能性需求

## 7.1 性能要求

• 并发能力：支持≥20路会话同时在线，高峰期可弹性扩展

• 响应速度：AI首响时间≤1秒，常规查询响应≤2秒，客服分配通知≤10秒

• 可用性：系统可靠性≥99.9%，支持故障自动切换

## 7.2 安全合规

• 数据安全：客户对话数据加密存储，保留期限≥2年

• 隐私合规：符合《个人信息保护法》，敏感信息（手机号、身份证号）脱敏展示

• 权限控制：基于角色的细粒度权限（客服仅可见自己会话，主管可见团队，总监可见全部）

• 操作审计：客服分配、会话记录、数据导出等操作完整留痕

## 7.3 部署方式

• 私有化部署

**【产品化预留】** 当前采用私有化部署，架构层预留SaaS多租户路由能力（租户ID透传、数据隔离字段）。未来如需转SaaS，仅需启用租户路由层+配置中心即可。
